From Landing to Action

Stage 1 · Landing
1
Landing Page — First Impression
Entry Point
User arrives via search, referral, social share, or partner link. The landing page establishes the platform's promise in one screen: honest grant readiness scoring — not a calculator, a preparation platform.
Hero copy and value proposition above the fold
Social proof: score share examples, beta partner badge (United Way)
Single primary CTA: "Score Your Application"
Secondary navigation: Grant Type Library · Funder Directory · Compliance Calendar
No login required to begin — frictionless entry into scoring flow
Stage 2 · Intelligence Input
2
Grantor Intelligence — Module 01
Input Layer
Before the application is scored, the user establishes funder context. This is the single highest-leverage input — it transforms a generic score into a calibrated one. User enters funder information via a structured, 5-field form.
Funder mission statement and stated values (free text)
Grant type selector: operations / program / capital / capacity building / emergency / seed / prize / corporate
Funding priorities and geographic focus
Eligibility restrictions (optional)
Average grant size range (optional — used for budget calibration)
3
Organizational Readiness Assessment — Module 02
Input Layer
The user self-assesses their organization across the dimensions funders evaluate before reading an application. Structured checklist format — fast to complete, honest in output.
IRS compliance: 990 current, determination letter on file
State registration: charitable solicitation active
Financial documents: audit, review, or compilation on hand
Board governance: roster current, COI policy adopted
Digital presence: website active, contact info current
Marketing presence: social platforms active, photos/videos published, press or testimonials
Program documentation: logic model and evaluation framework on file
4
Application Text Input — Module 03 Setup
Input Layer
User pastes or uploads the grant application narrative. Each scored dimension has a labeled text field or the user submits the full narrative for AI parsing. Clear character guidance per field ensures input quality.
Statement of Need field
Goals and Objectives field
Program Design and Methodology field
Evaluation Plan field
Budget Narrative field
Organizational Capacity field
Progress indicator: shows completion before scoring is triggered
Stage 3 · Scoring Engine
5
Funder Fit Score Generated — Module 01 Output
Engine
The engine generates a Funder Fit Score (0–100) based on alignment between the organization's profile and the grantor intelligence inputs. This score runs before the application score and gates the scoring experience.
Fit Score displayed prominently with plain-language interpretation
If Fit Score is below 60: advisory surfaced — "Review funder alignment before proceeding"
User may proceed or return to revise grantor inputs
Fit Score remains visible throughout the scoring session as a context anchor
6
Application Scored — Module 03 Engine
Engine
The AI scoring engine evaluates all seven application dimensions using weighted criteria calibrated to the grant type and funder profile. Scores are not inflated. Each dimension receives a sub-score and specific, actionable feedback.
Seven dimensions scored with individual weights applied
Each dimension: sub-score + 2–3 sentence specific feedback (what works, what is missing, what to fix)
Readiness profile gaps incorporated into overall score weighting
Composite score calculated with breakdown visible
Stage 4 · Results
7
Results Dashboard — Score + Full Breakdown
Output
The results page is the core product experience. It must be immediately readable, honest about the score, and specific enough that the user knows exactly what to do next — whether their score is 92 or 41.
Composite score displayed large — no softening, no rounding up
Funder Fit Score displayed alongside application score
Visual dimension breakdown (radar or bar chart) showing relative strengths and weaknesses
Per-dimension feedback: score + specific 2–3 sentence advisory
Readiness gaps surfaced with priority flags (Critical / Significant / Developmental)
Score band interpretation shown clearly (see bands below)
85–100
Competitive

Submit with confidence. Address any flagged items before deadline.

70–84
Viable

Address all flagged gaps before submitting. Competitive with revision.

55–69
Marginal

Significant revision required. Consider whether to submit this cycle.

Below 55
Not Ready

Do not submit. See Readiness Path for structured next steps.

Stage 5 · Decision Branch
8
Score Gate — Ready to Apply?
Decision Point
Based on the composite score, the platform routes the user to one of two primary tracks. This is not a wall — it is honest guidance. Users scoring below 70 are not blocked; they are redirected toward building the capability to succeed.
✓ Score ≥ 70 — Ready Track

Proceed to Application

User receives a prioritized action checklist of remaining gaps to address before submission. Grant Prep Checklist (Module 04) surfaced with completion status. Compliance Calendar link provided for filing deadline awareness. Share Your Score module (Module 10) activated.

⚠ Score < 70 — Readiness Track

Build Before You Apply

User routed to the Readiness Path (Module 08) — a structured 90-day or 12-month action plan built from their specific gaps. Alternative grant types surfaced that match current readiness level. Compliance calendar embedded as milestones. Option to rescore after improvements made.

Stage 6 · Post-Score Actions
9
Grant Prep Checklist — Module 04
Action Output
Users on the Ready Track receive their Grant Prep Checklist with completion status pre-populated from their readiness assessment. Items marked missing are flagged by priority. Users can download or print the checklist.
Pre-populated with items marked complete / missing / unknown from readiness assessment
Priority flags: Critical (likely disqualifying if missing) / Significant / Developmental
Each item includes plain-language explanation of why funders require it
Downloadable PDF checklist for offline use
10
Share Your Score — Module 10
Viral Loop
After results, users can generate and share platform-ready social content built from their actual score data — not templates. This is the acquisition engine. Every shared score drives new users to the platform.
LinkedIn post — professional, funder-facing, credibility-building
Facebook post — community and nonprofit group optimized
Instagram caption with relevant hashtags
X/Twitter — punchy hook with score and one key result
All content personalized to actual dimension scores — not generic "I scored 84" copy
Standing Resources — Accessible at Any Point in the Flow
Module 05

Grant Type Library

Every major grant type explained — requirements, typical seasons, realistic timelines. Accessible from navigation at any point. Educates users before they enter the scoring flow.

Module 06

Grant Season Calendar

Federal and major foundation grant cycles by type and funding area. Filterable. Helps users plan 90–180 days ahead rather than react to open windows.

Module 07

Local Funder Directory

Curated directory of local and regional funders with grant seasons, award ranges, and contact links. NC-anchored in V2. The feature that drives return visits.

Module 08

Readiness Path

Structured 90-day or 12-month action plan for organizations not yet competitive. Triggered by score gate but also accessible directly from navigation.

Module 09

Compliance Calendar

990 deadlines, state charitable solicitation renewals, and governance filing requirements. Static resource page plus tool-integrated milestone flags.

Entry Point

Re-Score Anytime

Users who completed the Readiness Path can return and rescore. Platform recognizes returning users and surfaces prior score for comparison. Progress is the product.
